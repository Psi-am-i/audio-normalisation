# Building the distributable app

This produces `Normalizer.app` — a self-contained, double-clickable macOS app
that bundles Python, the normalizer, and a static **ffmpeg**. Recipients install
nothing. Double-clicking opens a Terminal window with the interactive flow
(source folder → destination folder → AIFF/FLAC).

Only the **manual** mode is shipped. The autowatch daemon (`watcher.py`) stays in
the repo for your own use and is excluded from the app.

## Build it

From the repo root:

```bash
packaging/build_app.sh
```

Outputs:
- `packaging/dist/Normalizer.app` — the app
- `packaging/dist/Normalizer.zip` — zip this to people (always send the zip, not
  the raw `.app`, so macOS keeps the executable bits)

### ffmpeg source & licensing

A **static** ffmpeg binary is required — Homebrew's `ffmpeg` links against many
dylibs and is not portable. The build scripts get one, in priority order:

1. `$FFMPEG_STATIC` — a static ffmpeg binary you already have:
   ```bash
   FFMPEG_STATIC=/path/to/static/ffmpeg packaging/build_app.sh
   ```
2. Otherwise they download the self-contained static build from
   [evermeet.cx](https://evermeet.cx/ffmpeg/), which is **GPLv3**
   (`--enable-gpl --enable-version3`) and redistributable.

**Licensing:** the build **refuses** any ffmpeg built `--enable-nonfree` — those
binaries cannot legally be redistributed. Because the app calls ffmpeg as a
separate subprocess (not linked), the app's own code is not a derivative work of
ffmpeg. Each zip includes a `licenses/` folder with the full GPLv3 text and an
attribution NOTICE (version + source links), generated by `lib_licenses.sh`.

**Architecture:** the evermeet build is **x86_64**. The PyInstaller front-end is
built for the host arch. So on an Apple Silicon Mac you get an arm64 app that
runs its bundled x86_64 ffmpeg via **Rosetta 2** (present on essentially all
Apple Silicon Macs; auto-installs on first use). The app runs on:

| Recipient        | Works? |
|------------------|--------|
| Apple Silicon    | ✅ (ffmpeg via Rosetta 2) |
| Intel            | ✅ only if the front-end is also x86_64 — build under `arch -x86_64` or on an Intel Mac |

For most modern (Apple Silicon) recipients the default build is fine.

## Gatekeeper (unsigned app)

The app is **not code-signed or notarized**, so on another Mac Gatekeeper will
block the first launch. The recipient does this once:

> Right-click (or Control-click) `Normalizer.app` → **Open** → **Open**.

After that it launches normally by double-click.

To remove the warning entirely you'd need an Apple Developer ID ($99/yr) to sign
and notarize — out of scope here. If you get one, add the identity to
`packaging/normalizer.spec` (`codesign_identity`) and run `xcrun notarytool`.

## How the app is structured

```
Normalizer.app/
  Contents/
    Info.plist
    MacOS/Normalizer          # launcher: opens Terminal, runs the CLI
    Resources/normalizer-cli  # PyInstaller onefile binary (python + tqdm + ffmpeg)
```

At runtime `normalizer.resolve_ffmpeg()` finds the bundled ffmpeg via
`sys._MEIPASS` before falling back to `$FFMPEG_BINARY`, `PATH`, or Homebrew.
